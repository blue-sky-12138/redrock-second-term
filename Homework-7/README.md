## 项目结构

1.workloadManager为请求总管理器，用于进行转发请求到对应的子服务器，并实现简单的负载均衡和故障转移，但还不会判断是否故障(目前没有探索出方案)，只会绕过故障的服务器。(Ngnix还不会用= =)

2.usersCenter为已分离出的服务，用于提供用户登录认证，用户注册，更新用户资料。(然而由于部分功能中编写了有关检测用户是否存在的函数，目前并没有完全分离)

3.三个以Routers作为后缀的文件夹中是分理出的用户、视频、文件路由服务。



## 其他

1.由于没有多个服务器用来测试，这里每个子服务路由只是改了运行的端口。因此转发功能才会直接使用原HTTP请求的path。

2.子服务对于捕获的请求应该要鉴权，但还没想到很好地解决方案。

3.由于项目保存文件和我获取文件的路径问题，目前无法使用fileRouters的功能。(如果将其ctrl+c/v到总项目外应该能解决)
